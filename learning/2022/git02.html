Git基本操作指令

2022-10-21

在基本操作中會搭配圖形介面（Source Tree）一起使用，圖形介面的好處是可以少打一些字，相較於終端命令來說也較為清楚直觀。

操作指令

● Git初始化：git init

如果想要對開發的專案進行版本控制，可以使用這個指令初始化目錄，讓Git開始對這個目錄進行版控。
如果是利用Source Tree參考如附圖：

● 查看目錄狀態：git status

如果想要查詢現在這個目錄的狀態，可以在該目錄下執行以上指令。
可以用於檢視在此目錄下有哪些新的/被修改的檔案，如果檔案狀態為 Untracked files 表示該檔案尚未被Git追蹤；如果檔案狀態由Untracked files變更為 new file 表示該檔案已被安置在暫存區。

● 將檔案加入暫存區：git add .

如果將新增的檔案加入暫存區可以透過上述指令或使用 git add --all

● 保存此次暫存結果：git commit -m "〈此次版本資訊〉"

將檔案加入暫存區是不夠的，如果要讓暫存區內的紀錄永久保存的話，還需透過以上指令，才能夠將暫存區的內容保存在倉庫（Repository）內。 一次的commit即可以視為一個"版本"。
