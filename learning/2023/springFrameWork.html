SpringBoot

2023-03-04

近年來越來越多的開發者採用微服務（Micro Services）方式進行程式開發。它是一種可以將應用程式拆解成各個細項服務的技術，讓各個服務可以專注於某些特定的功能。
微服務的優點在於：
1.耦合性低，不論在開發階段或是在部署程式階段都是獨立的，當一項微服務出現問題時，不會直接影響其他服務和程式的運作
2.程式碼簡單，每個微服務中都較為單純，盡可能的讓微服務只專一做一件事情
3.開發快速，微服務可以針對個別服務進行修正或是擴充新功能

而Spring FrameWork也算是目前Java工程師用來開發微服務中常被使用的框架之一，也是一種後端程式的框架。

什麼是Spring FrameWork

它是一個以Java為基礎的開放原始碼Web應用程式架構，在Spring框架下，包含許多子項目，日後有機會接觸其他項目的話會再分享。
而今日這篇的主角主要是Spring Boot。

什麼是Spring Boot

Spring Boot是基於Spring FrameWork 4.0 而衍生的，它主要為了簡化使用Spring框架的難度（建立、執行、除錯、部署等）以及繁雜的依賴配置。

舉例來說，假設現在需要開發一個Web Server的專案，可能會需要設定很多東西才能達到需求，但如果使用Spring Boot啟動專案就會非常的快速。

Spring Boot 簡化以及加快了 Java Web 的開發速度，開發者可以將更多注意力放在專案程式碼的開發上。

Spring Boot的優勢

1.可以快速構建項目，啟動快速，縮短開發的時間
2.可以獨立運行，無須外部依賴Servlet（可選擇內嵌Tomcat等Servlet容器，無須透過war檔的方式部署項目）
3.高度的自動化配置
4.容易管理各種依賴項目
5.減少程式碼的數量（行數等）

Spring Boot的缺點

1.可能不適用於大型項目開發
2.版本迭代速度快，有些第三方library改動大
3.自動配置，簡化了許多架設流程，雖然入門簡單，但底層因被封裝過，所以想要深入理解是有一定難度
